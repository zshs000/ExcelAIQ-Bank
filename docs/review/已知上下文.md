## 已确认误判

以下项经用户确认为误判，审查时不再报：

| ID | 原审查项 | 误判原因 |
|----|---------|---------|
| W1 | `@ConditionalOnProperty(matchIfMissing=true)` | 生产环境消费者必须默认注册，静默不注册比本地报错危险得多 |
| W2 | `JsonUtils.OBJECT_MAPPER` 非 volatile | `init()` 仅在 Spring 启动时调用一次，之后只读，容器初始化有内存屏障 |
| W3 | Redis `increment` + `expireAt` 竞态 | `INCR` 是原子操作，全局只有一个请求拿到 `count==1`，不存在并发竞态 |
| W4 | `AuthController` 缺少 `@RequestMapping` | 网关 `StripPrefix=1` 设计，加了反而路径冲突 |
| W5 | `@SneakyThrows` 滥用 | SDK IO 异常走 `@SneakyThrows` 冒泡，全局异常处理器兜底，业务校验已用 `BizException` |
| W6 | `QuestionExcelListener` 异常丢失 | `chunkConsumer` 抛异常时 `buffer.clear()` 不执行，数据仍在 buffer，整条导入已失败 |
