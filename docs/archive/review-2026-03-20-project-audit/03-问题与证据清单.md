# 问题与证据清单

## P0 / 高风险

### 1. 自动注册用户使用固定默认密码，可被手机号 + 弱口令直接撞入

- 现象
  - `user` 服务给验证码自动注册的新用户统一设置默认密码 `123456`
  - `auth` 服务又支持手机号 + 密码直接登录
- 代码证据
  - `eaqb-user/eaqb.user.biz/src/main/java/com/zhoushuo/eaqb/user/biz/service/impl/UserServiceImpl.java:49-50`
  - `eaqb-user/eaqb.user.biz/src/main/java/com/zhoushuo/eaqb/user/biz/service/impl/UserServiceImpl.java:269-289`
  - `eaqb-auth/src/main/java/com/zhoushuo/eaqb/auth/service/strategy/impl/PasswordLoginStrategy.java:35-54`
- 影响
  - 只要用户完成过验证码登录但未改密，其账号就天然存在已知默认口令
  - 手机号往往可被猜测或枚举，这会把默认口令变成可利用入口
- 建议
  - 取消固定默认密码
  - 若必须兼容密码登录，新增“密码未初始化”状态，要求用户首次显式设置密码
  - 更稳的做法是把验证码登录和密码登录解耦，不给自动注册用户默认口令

### 2. 业务服务直接信任请求头 `userId`，旁路访问时可伪造身份

- 现象
  - 网关鉴权后会把 `userId` 写进请求头
  - 下游服务看到请求头就直接写入 `ThreadLocal`
  - 业务方法普遍直接读取 `LoginUserContextHolder.getUserId()`
- 代码证据
  - `eaqb-gateway/src/main/java/com/zhoushuo/eaqb/gateway/filter/TrustedInternalCallGatewayFilter.java`
  - `eaqb-framework/eaqb-spring-boot-starter-biz-context/src/main/java/com/zhoushuo/framework/biz/context/filter/TrustedInternalRequestFilter.java`
  - `eaqb-framework/eaqb-spring-boot-starter-biz-context/src/main/java/com/zhoushuo/framework/biz/context/config/ContextAutoConfiguration.java:12-16`
  - `eaqb-question-bank/eaqb-question-bank-biz/src/main/java/com/zhoushuo/eaqb/question/bank/biz/service/impl/QuestionServiceImpl.java:631-641`
  - `eaqb-user/eaqb.user.biz/src/main/java/com/zhoushuo/eaqb/user/biz/service/impl/UserServiceImpl.java:236-246`
- 影响
  - 如果 `question-bank`、`user`、`excel-parser` 等服务端口被直接访问，攻击者可以自带任意 `userId`
  - 这会导致越权查改题目、越权改密、越权读取业务数据
- 建议
  - 最少要限制业务服务只允许网关或内网访问
  - 更稳妥的是服务内也校验来自网关的可信凭证，或使用服务间签名/内部 token

  - 即使数据库密码做了 Druid 解密，解密公钥也与密文一起提交，防护价值很有限
- 建议

  - 改为环境变量、外部配置中心密钥、`.env.example` 或占位配置
  - 生成器配置也不要提交真实密码

### 5. 网关和上下文过滤器保留大量调试日志

- 代码证据
  - `eaqb-gateway/src/main/java/com/zhoushuo/eaqb/gateway/filter/TrustedInternalCallGatewayFilter.java`
  - `eaqb-framework/eaqb-spring-boot-starter-biz-context/src/main/java/com/zhoushuo/framework/biz/context/filter/TrustedInternalRequestFilter.java`
